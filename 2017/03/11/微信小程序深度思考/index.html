微信小程序深度思考

微信小程序深度思考

  1. 设计小程序的思维转换

    小程序是用户体验介于公众号和app之间的一款产品

    • 森林与河流的区别
      • 小程序： 森林模型 （所有树木都会整齐生长）
      • 公众号之类的具有中心流量产品： 河流模型（大鱼活在大河里，小鱼生活在小溪里，没办法与大鱼竞争）
    • 重服务与重内容之间的区别
      • 小程序： 重服务，随机性强，在适合的场景下想到并且使用然后离开
      • 公众号： 重内容，提供足够好的内容或者能吸引到用户的内容，然后让用户长期存留
    • 服务用户与沉淀用户之间的区别
      • 小程序： 服务用户，让用户在特定的场景下想到某个应用 （场景）
      • 公众号： 沉淀用户，以能够吸引用户的内容来达到用户留存沉淀（拉新+留存+激活）
    • 发掘用户对小程序的需求
      1. 功能需求尽量单一，不适合复杂项目
      2. 满足随机需求（比如：坐地铁时候、点餐的时候、想骑车的时候等）固定需求不太适合
      3. 高频需求
      4. 碎片化需求，需要用户停留很长时间的产品不太适合
      5. 长尾需求，不想开发app就可以做小程序，或者说没有必要做app的场景，比如 换轮胎之类的
      6. 线下场景，等车、点餐、商铺出租、无人购物等等

  2. 几类主要的适用场景

    • 工具
    • 服务
    • 内容
    • 电商
      1. 找到小程序在微信社交场景下的优势，如送东西给别人、拼团、众筹等有创意的需求；
      2. 电商入口，例如用照片搜索商品，自动带扫描商品二维码判断商品真伪，拍张照片根据我的脸型为我选一副眼镜之类的等等；
    • 社交／社群
      • 找到微信还没有涵盖服务的社交场景，例如婚恋、兴趣之类的小程序
    • 游戏（暂未开放）

  3. 小程序与产品矩阵

    产品矩阵是公司为了达到某一战略目的而整体铺开的一些产品

    • 常见的产品矩阵包括：

      1. 多个微信小程序
      2. 一个或者多个app
      3. pc网站／微站（即手机网站）
      4. 公众号

      因为小程序能够满足一些特定的用户需求，所以企业能将小程序添加到自己的产品矩阵里

  4. 小程序的运营与推广

    1. 借助小程序搜索
    2. 借助人际传播（关键点）
    3. 地推和线下扫码（关键点）
    4. 结合公众号运营

MYSQL忘记密码了怎么重置密码?

mysql忘记密码了怎么重置密码?

  1. MySQL版本8.0.4之后修改密码

    ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '新密码';

  2. 之前的重置方法请自行百度

    具体原因是MySQL 8.04前，执行1,2都可以：

    1. SET PASSWORD=PASSWORD('[修改的密码]');
2. update user set password=password('123456') where user='root';

    但是MySQL8.0.4开始，这样默认是不行的。因为之前，MySQL的密码认证插件是:

    mysql_native_password

    而现在使用的是:

    caching_sha2_password

微信小程序深度思考

微信小程序深度思考

  1. 设计小程序的思维转换

    小程序是用户体验介于公众号和app之间的一款产品

    • 森林与河流的区别
      • 小程序： 森林模型 （所有树木都会整齐生长）
      • 公众号之类的具有中心流量产品： 河流模型（大鱼活在大河里，小鱼生活在小溪里，没办法与大鱼竞争）
    • 重服务与重内容之间的区别
      • 小程序： 重服务，随机性强，在适合的场景下想到并且使用然后离开
      • 公众号： 重内容，提供足够好的内容或者能吸引到用户的内容，然后让用户长期存留
    • 服务用户与沉淀用户之间的区别
      • 小程序： 服务用户，让用户在特定的场景下想到某个应用 （场景）
      • 公众号： 沉淀用户，以能够吸引用户的内容来达到用户留存沉淀（拉新+留存+激活）
    • 发掘用户对小程序的需求
      1. 功能需求尽量单一，不适合复杂项目
      2. 满足随机需求（比如：坐地铁时候、点餐的时候、想骑车的时候等）固定需求不太适合
      3. 高频需求
      4. 碎片化需求，需要用户停留很长时间的产品不太适合
      5. 长尾需求，不想开发app就可以做小程序，或者说没有必要做app的场景，比如 换轮胎之类的
      6. 线下场景，等车、点餐、商铺出租、无人购物等等
